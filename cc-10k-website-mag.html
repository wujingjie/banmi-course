用 Claude Code 打造价值 $10,000 的网站

📺 Metics Media⏱️ 26:09🏷️ Claude Code 系列 · 教程精读▶ 原视频
作者用 Claude Code 给一家西雅图餐厅做了一个看起来像自由职业者收费 $10,000 的落地页。核心套路：给 Claude Code 装两个设计 skill、写一条会"反问澄清问题"的 prompt、再对照一张「$10k 网站 8 要素清单」逐项自评打磨。本页是教程型精读：每步给「可复制操作 + 🧠费曼为什么 + 预期结果 + 踩坑」。

◆一分钟速览▾

作者（Metics Media）演示用 Claude Code 桌面版给一家西雅图餐厅做高端落地页。不手写设计，而是：装两个设计 skill → 写一条带"请反问我"的 prompt → 用一张 8 要素清单反复自评打磨 → 配 AI 生成的图/视频 → 上线到真实域名。最终成品暗黑、沉稳、有杂志感和光标交互，访客看不出是 AI 做的。

诚实提醒

"$10,000" 是营销话术（卖的是"专业观感"）。视频里出现的 Hostinger / 11 Labs 都带 affiliate 折扣链接（带货环节），托管和素材工具都可替换，不影响 Claude Code 的核心流程。

🎯先定义：$10,000 网站 = 8 件事▾

动手前作者先问 Claude："什么把 $10,000 的网站和 $200 的网站区分开？"得到 8 个要素，归为三组——后面每一步都在凑这 8 项：

  • 品味（taste）：① Point of view 立场 / ② Typography 排版 / ③ Color 配色
  • 内容（substance）：④ Hierarchy 层级 / ⑤ Imagery 图像 / ⑥ Motion 动效
  • 体感（felt quality）：⑦ Mobile 移动端 / ⑧ 隐形质感（让站点"又快又完整"的东西）

为什么先列清单？因为有了明确标准，后面才能让 Claude 对照自评打分（第 ④、⑦ 步都靠它）。这是把"好不好看"从主观变成可检查项的关键。

🧰前置条件 / 环境准备▾

  • Claude 账号：视频用 Pro 计划（录制时约 $20/月），足够本教程。
  • 一台电脑：Mac 直接用；Windows 按视频描述走 WSL。
  • 两个设计 skill：front-end design（Anthropic 官方）+ UI UX Pro Max（社区），链接在视频描述里，都免费。
  • （素材，可选）：ChatGPT / 11 Labs 生成图与视频；21st.dev 抓组件。
  • （上线，可选）：任意静态托管 + 域名（视频用 Hostinger，可换 Netlify/Cloudflare Pages 等）。

01安装 Claude Code 并进入 code 模式▾

🎯 这步要干嘛

装好 Claude Code 桌面版、登录、切到 code 模式，并给项目建一个独立文件夹。

📌 操作 / 命令

# 1. 打开 claude.com（用描述里的链接），选 Pro 计划注册
# 2. 左下角下载图标 → Claude Code 区 → 下载桌面版 → 运行安装
# 3. 打开桌面 App 登录
# 4. 切到 code 模式：左上角切 chat→code，或按 Cmd+3 (Mac) / Ctrl+3 (Win)
# 5. 在电脑上新建一个空文件夹，点聊天框旁的文件夹按钮选中它作为工作区
🧠 费曼 · 为什么这么做

Claude Code 是跑在桌面 App 里的 AI 编程助手——你用自然语言描述网站，它边写代码你边看。给它一个独立文件夹当工作区，是因为它要在里面读写真实文件，这是它能"端到端建站并打包上线"的前提。注意作者提醒：Claude 更新很频繁，你看到的界面可能跟视频不完全一样，抓概念别抠按钮位置。

✅ 预期结果 / 怎么验证

进入 code 模式、选好项目文件夹后即就绪。

⚠️ 常见坑

Windows 直接装容易踩环境坑，按视频走 WSL 更稳。

02装两个设计 skill（这步是分水岭）▾

🎯 这步要干嘛

装上 front-end design + UI UX Pro Max 两个 skill，再切到 auto 模式。

📌 操作 / 命令

# skill 1 · front-end design（Anthropic 官方，后台静默生效）
在 Claude Code 里粘贴该 skill 链接 + 消息：install this skill
# 允许编辑，选全局安装(globally)

# skill 2 · UI UX Pro Max（社区，需要时用 / 主动调用）
粘贴该 plugin 链接 + 消息：install this plugin using npm
# “using npm”这措辞很重要，能让它走终端干净安装；允许编辑、全局安装

# 最后：左下角模式选择器 → 切到 auto 模式（让它不必每步停下问权限）
🧠 费曼 · 为什么这么做

默认 Claude Code 很会写代码，但设计输出只是"还行"——爱用同样的字体、配色、布局，这就是多数 AI 网站显得 generic 的原因。skill 本质是给它的"专项说明书"：front-end design 在后台运行，封杀网上最烂大街的字体、把它推向大胆的设计方向；UI UX Pro Max 带 57 种 UI 风格 / 95 套配色 / 56 组字体搭配，需要大改造时用 / 主动召唤。装上这俩，才从"能用"跨到"惊艳"。

✅ 预期结果 / 怎么验证

输入 / 能看到 UI UX Pro Max 可调用；两个 skill 都装到全局后，之后每个项目都自动带审美。

⚠️ 常见坑

第二个 skill 必须用 "install this plugin using npm" 这个说法，措辞不对会装不干净。

03写 prompt + 让它"反问澄清问题"▾

🎯 这步要干嘛

用 /UI-UX-Pro-Max 起头写简报，结尾加"ask me clarifying questions"，回答它的问题来定方向。

📌 操作 / 命令

/UI-UX-Pro-Max
<在这里写你的简报：什么生意、想要的氛围、要哪些版块……>
Ask me clarifying questions.

# 不会用文字描述审美？用 references 抄近路：
# 去 Dribbble / Awwwards / Pinterest 找 3-5 个你喜欢的站，截图丢给 Claude
🧠 费曼 · 为什么这么做

最后那句 "Ask me clarifying questions" 是灵魂——它让 Claude 先停下来思考、反问，而不是闷头就建。视频里它回了 7 个问题（餐厅名、风格方向、要哪些版块、谁写文案、技术栈、动效程度等），还给出3 个风格方向让你选（曼哈顿牛排馆暗黑风 / 太平洋西北现代风 / 老派 chop house）。你在这里给的答案=最终整个网站，答得越具体，后面越不用跟它返工。描述不出审美时，截图比文字管用得多。

✅ 预期结果 / 怎么验证

Claude 反问 7 个问题 + 给 3 个风格方向；你认真回答后它才开始建。

⚠️ 常见坑

别写"build me a coffee website"这种懒 prompt——那必然 generic。

04生成 + 对照 8 要素清单自评打分▾

🎯 这步要干嘛

让它建站，然后把 8 要素清单贴回去，让 Claude 诚实地逐项打分。

📌 操作 / 命令

# 它会先"读简报想 3 分钟"再写 4-5 分钟代码，然后 reveal
# 自评：把 $10k 清单贴给它
Here is the $10k checklist. Where does this site land against each of these criteria? Be honest.
🧠 费曼 · 为什么这么做

生成只是开始，自评打分才是把质量拉上去的闭环。让 Claude 对照清单逐项说"哪项强、哪项一般、哪项缺"——视频里结果是：立场/配色/层级 strong，排版 mixed（因为用了烂大街的 Inter 字体），图像/动效/移动端/隐形质感都 mixed。知道差在哪，才知道接下来修什么。

✅ 预期结果 / 怎么验证

Claude 逐项给出 strong / mixed / missing 评价，你就有了一张"待办清单"。

05配图像 / 视频素材▾

🎯 这步要干嘛

用自有素材，或用 AI 生成图与视频，再丢进项目文件夹。

📌 操作 / 命令

# 让 Claude 替你写图像生成 prompt（它了解你的项目和风格）
Write me an image generation prompt for the hero, matching our design aesthetic.
# 把它给的 prompt 粘到 ChatGPT 图像模型 / 11 Labs 里生成
# 需要视频：11 Labs 里用该图做首帧 → Google VO3.1 生成视频 → Topaz 放大
# 拿到素材后丢进项目文件夹，告诉 Claude 文件在哪、怎么用
🧠 费曼 · 为什么这么做

图像是 Claude 自己搞不定的一项，得你喂。最好是为品牌定制的拍摄/设计；没预算就用 AI 生成——现在的工具用对了看不出来。关键技巧：别自己写图像 prompt，让 Claude 写——它懂你的项目和既有审美，生成的素材第一次就对味。ChatGPT 出图快但不能出视频，要视频就用 11 Labs 这类聚合器一站式跑（出图→VO3.1 转视频→Topaz 放大）。

✅ 预期结果 / 怎么验证

素材放进项目文件夹、告诉 Claude 后，它就能把图/视频用进页面。

⚠️ 常见坑

11 Labs 是付费聚合工具（视频带 affiliate 折扣）——只是图省事，可换任意出图/出视频方案。

06用 21st.dev 组件做英雄区效果▾

🎯 这步要干嘛

从 21st.dev 抓一个喜欢的组件，让 Claude 在不破坏架构的前提下用进来。

📌 操作 / 命令

# 在 21st.dev 找到喜欢的滚动/图像组件 → 点 "copy prompt"
# 回 Claude Code 粘贴该 prompt + 你生成的素材：
Use this component to build out the hero section.
🧠 费曼 · 为什么这么做

21st.dev 是一堆真实开发者做的免费组件库（动效、滚动效果、按钮……），每个能"copy prompt"直接让 Claude 复刻。值得学的是 Claude 的架构自我保护：作者选的组件基于 React，而本项目是简单静态站，Claude 会拒绝直接塞 React（那会改变整个项目结构），改用纯 HTML 重写同样效果。它不会因为你要就乱来。第一次没成功也别慌——视频里 Claude 自己打开浏览器检查实时页面，三层排查后修好了滚动效果。

✅ 预期结果 / 怎么验证

得到一个电影感的滚动英雄区；遇 bug 就告诉它"现象+你期望的结果"，它会继续深挖直到修好。

⚠️ 常见坑

若你的项目本就复杂（带登录/支付/数据库），Claude 反而会同意上 React——它按项目实际需要判断。

07批量改进（按"意图"而非"细节"下令）▾

🎯 这步要干嘛

把待改项打包成一批，用"我要什么感觉"来下令，而不是逐条指定怎么改。

📌 操作 / 命令

We need more handcrafted micro-interactions. The lower sections feel a bit generic. We don't need to make them busier, just more expensive.
🧠 费曼 · 为什么这么做

两个要点：① 批量改、别一条条改——一次让它提一批修复（视频里一口气加了全站颗粒质感、版块间的黄铜细线、预订区的余烬微光、标题逐词浮现等），省 token、结果更协调；② 下令讲意图不讲细节——"更精致、不要更花哨"这种"感觉"，Claude 把意图翻译成具体实现，比你写规格说明更强。

✅ 预期结果 / 怎么验证

Claude 一次返回一批改动，你一次批准，全站质感整体提升。

08逐区加光标 / 动效（最显贵的一层）▾

🎯 这步要干嘛

滚一遍页面，挑出还显"平"的版块，每区给一个克制的光标交互或动效。

📌 操作 / 命令

# 对每个还很平的版块，给一条具体动效指令，例如：
Make the embers move like fire. Have them react to the cursor as it moves around the page.
Add elegant micro-movements and cursor interactions here; this section feels static. Make it subtle and refined.
🧠 费曼 · 为什么这么做

这层是"看起来贵"和"用起来贵"的分界。Claude 看不出哪里平——只有你能"感觉"到，所以这一遍要你自己开车：逐区找平淡处，每区只加一个克制的交互。视频里余烬随光标像火一样被扰动、故事区有跟随光标的烛光光晕（第一版太明显，作者让它"更微妙"，于是改成滞后跟随）。一次一区、都保持 subtle。

✅ 预期结果 / 怎么验证

每个平淡版块加完一个微交互后，页面"活"起来但不吵闹。

⚠️ 常见坑

动效太抢戏就让它"more subtle / refined"，宁可含蓄。

09打磨文案 + 换掉烂大街字体▾

🎯 这步要干嘛

让文案走"克制"路线，并把 AI 标志性的 Inter 字体换掉。

📌 操作 / 命令

# 文案：克制、少形容词（如 "Six dishes, one fire" 这种）
# 字体：看到 Inter 就换
This uses Inter, which screams AI. Swap it for something better.  # 视频里换成了 Geist
🧠 费曼 · 为什么这么做

两个一眼出戏的 AI 痕迹：① 文案爱过度解释、堆形容词——好文案是克制（"Six dishes, one fire" 这种），front-end design skill 会帮 Claude 写这种感官化的克制文字；② Inter 字体在 AI 站上烂大街，一看就"AI 做的"，换成 Geist 这类立刻不一样。

✅ 预期结果 / 怎么验证

文案变克制、字体换掉后，整体读感更像真人设计师出品。

10移动端专门过一遍▾

🎯 这步要干嘛

让 Claude 为手机单独设计一遍，而不是只把桌面版缩小。

📌 操作 / 命令

Do a dedicated pass on the mobile version: what should be hidden on small screens, what should be tighter, and what should resize.
🧠 费曼 · 为什么这么做

Claude 默认会做响应式（能在手机上显示），但"响应式 ≠ 为手机设计"。手机访问量远大于桌面，如果手机版只是桌面版压扁，用户能感觉到。所以要明确让它为小屏做取舍。视频里：导航在手机折叠、字标间距收紧、按钮换小尺寸变体。

✅ 预期结果 / 怎么验证

手机上导航收起、间距更紧、按钮更合手——而不是缩小版桌面页。

11上线到真实域名▾

🎯 这步要干嘛

让 Claude 打包静态文件，传到托管、绑定域名。注意压缩的是"文件夹里的文件"。

📌 操作 / 命令

# 1. 托管选型：纯静态站(无登录/支付/数据库)选 premium；
#    带后端(react/node)选 business —— 想想第 6 步 Claude 拒不拒 React 就知道选哪档
# 2. 套餐选 12 个月（拿第一年免费域名），搜并注册你的域名
# 3. 关键：压缩的是【文件夹里的文件】不是【整个文件夹】！
#    进项目文件夹 → 全选里面的 index.html + 资源(Cmd/Ctrl+A) → 压缩成 zip
#    （直接压整个文件夹的话，站会被解到深一层目录，域名打开是空白页）
# 4. 托管引导选 migrate existing site → upload backup files → 选域名 → 拖 zip → next → submit
🧠 费曼 · 为什么这么做

静态站（纯 HTML/CSS/JS、无后端）可直接丢进托管的目录就能跑，所以让 Claude "打包成可静态托管的文件"。最容易翻车的是压缩：要压文件夹里的文件，不是文件夹本身——否则解压后多套一层目录，域名加载空白页。托管商不重要，premium 够静态站用；只有 Claude 给你建了真后端才需要 business。

✅ 预期结果 / 怎么验证

上传、submit 后点你的域名，就能看到上线的网站。

⚠️ 常见坑

视频这段是 Hostinger 赞助/带货（含 affiliate 折扣）；本质只是"静态文件丢上托管"，换任意服务商都行。

🧠费曼总纲：把 Claude Code 当"会自评的精装队"▾

整个流程像装修一套样板房：

  • Claude Code = 施工队：手巧能盖，但默认没什么审美。
  • 两个 skill = 请来的设计总监：front-end design 是"高级感标准"（封杀俗气字体），UI UX Pro Max 是"风格/配色素材库"。装上就从包工头升级成有总监盯着的精装团队。
  • 8 要素清单 = 验收标准：先把"什么叫装得好"写成可勾选项，后面才能让总监自己照单验收。
  • 带"反问"的 prompt = 需求交底会：你描述想要的氛围，让总监反问把需求钉死，再开工。
  • 自评打分 + 批量改进 = 验收返工：照清单找毛病、一批批返工，这是质感来源。
  • 逐区光标动效 = 软装灯光：硬装好后，你亲自挑平淡处点缀——这步机器替不了你的"手感"。
  • 打包上线 = 样板房搬到真地址：静态文件整体搬运、挂上门牌（域名）开门迎客。

内核：你负责把"想要的感觉"和"验收标准"讲清楚，Claude（带审美 skill）负责把上千个实现决策做对、并照标准自评返工。

🎁最终成品 / 验收▾

  • ✅ 一个暗黑沉稳、有杂志感的餐厅落地页，命中 8 要素清单全部项；
  • ✅ 电影感滚动英雄区 + 全站颗粒/黄铜细线/逐词浮现等批量打磨；
  • ✅ 余烬随光标扰动、烛光光晕跟随等逐区微交互；
  • ✅ 专门设计过的移动端；换掉了 Inter；
  • ✅ 打包上线在自己的域名上（视频里全年约 $43 托管+域名，加 Claude Pro 约 $60 做出 $10k 观感）。
怎么确认达成

手机和电脑各开一次域名：动效流畅、光标交互生效、小屏不塌、文案克制、看不出 AI——就算过关。

✅6 条要点▾
