用 Claude Code 打造价值 $10,000 的网站

📺 Metics Media⏱️ 26:09🏷️ Claude Code 系列 · 教程精读▶ 原视频
一条 prompt + 两个 design skill，让 Claude Code 从"能写代码"升级成"有审美的设计师"——做出带渐入动效、咖啡豆光标物理交互的高端落地页，再上线到真实域名。本页是教程型精读：每一步都给「可复制操作 + 🧠 费曼为什么 + 预期结果 + 踩坑」，照着做就能复现。

◆一分钟速览▾

作者用 Claude Code 给一个虚构高端咖啡品牌 Lumina 做了一个落地页：英雄区渐入、咖啡豆在背景漂浮、鼠标移动时豆子像在液体里一样被推开、菜单卡片悬停时抬起倾斜。全程没手写设计，只用自然语言描述"想要的感觉"，Claude Code 生成全部 HTML/CSS/JS。

做完你会得到

一个看起来像自由职业开发者收费 $10,000 的网站，上线在你自己的域名上，访客看不出是 AI 做的。整个流程 = 装工具 → 加两个设计 skill → 写一条好 prompt → 让它自审打磨 → 加动效 → 上线。

诚实提醒

"$10,000" 是营销话术（卖的是"专业观感"，不是真有人付这价）；视频结尾的 Hostinger 是带货赞助（含 affiliate 折扣链接），托管/域名你完全可以换任意服务商，不影响前面的 Claude Code 部分。

◆🧰 前置条件 / 环境准备▾

  • 一台电脑：Mac 直接用终端；Windows 建议用 WSL（视频描述里有教程链接）。
  • Claude 账号：有 Pro / Max 订阅则 Claude Code 直接包含；没有可以走 API 充值。
  • 两个 design skill 文件：design review skill + aesthetic direction skill（视频描述里有下载链接，本质是两个 markdown 文件）。
  • （上线用，可选）：任意静态托管 + 域名（视频用 Hostinger，可替换）。

◆成品长啥样▾

这就是目标形态：editorial（杂志感）的高端排版 + 克制的配色 + 有目的的动效。下面把它一步步拆开复现。

01安装 Claude Code▾

🎯 这步要干嘛

在本机全局装好 Claude Code 并登录，进入交互界面。

📌 操作 / 命令

# 1. 打开 claude.com/claude-code 复制官方安装命令，在终端执行
# 2. 安装完成后启动：
claude
# 3. 首次运行会要求登录，用已有 Claude 账号（Pro/Max 已包含）
🧠 费曼 · 为什么这么做

Claude Code 是一个跑在终端里的 AI 编程 agent——它能读写你项目里的真实文件、执行命令、看自己的运行结果。装在本机而不是网页，是因为它要直接操作你的文件系统和项目，这是它能"端到端建站并打包上线"的前提。

✅ 预期结果 / 怎么验证

终端输入 claude 后进入交互界面，登录成功即就绪。

⚠️ 常见坑

Windows 直接在 PowerShell 里装容易踩环境坑，按视频建议走 WSL 更稳。

02装两个设计 skill（关键升级）▾

🎯 这步要干嘛

给 Claude Code 装上 design review skill 和 aesthetic direction skill，让它"有审美 + 会自我评审"。

📌 操作 / 命令

# 在你的项目文件夹里建目录结构：
mkdir -p .claude/skills
# 把两个 skill 的 markdown 文件放进 .claude/skills/
# 验证是否加载：在 Claude Code 里输入
/skills
🧠 费曼 · 为什么这么做

默认的 Claude Code 很会写代码，但没有强烈的视觉审美。skill 本质是一份告诉它"该怎么表现"的 markdown：① design review skill 让它做完后给自己截图、像资深设计师一样评审间距/字体/对比度再改；② aesthetic skill 给它"品味"——一套关于"什么让界面显高级"的原则（留白要大方、配色要克制、动效要有目的）。这是整条流程从"能用"到"惊艳"的分水岭。

✅ 预期结果 / 怎么验证

输入 /skills 能看到两个 skill 都已 loaded；之后 Claude Code 会自动按这些原则工作。

⚠️ 常见坑

skill 文件要放对位置（项目根的 .claude/skills/ 下）才会被自动识别；放错目录则不生效。

03写对那一条 prompt▾

🎯 这步要干嘛

用一条"描述感觉而非功能"的 prompt 让 Claude Code 生成整站。

📌 操作 / 命令

Build a landing page for a premium coffee brand called Lumina.
The mood is warm, sophisticated, and calm.
Use a dark background with cream and gold accents.
Include a hero section with the product, a section about our sourcing,
a menu of signature drinks, and a contact section.
Make it feel editorial, like a high-end magazine.
🧠 费曼 · 为什么这么做

很多人写"build me a coffee website"这种懒 prompt，结果当然很 generic。秘诀是具体描述"感觉"而不是功能：给出心情（warm/sophisticated/calm）、配色（dark + cream + gold）、版块结构、以及整体调性（editorial 杂志感）。你不是在告诉它代码，而是在给它方向感——剩下成百上千个小决策（字体搭配、间距比例、配色比例）由它结合 aesthetic skill 自己拿主意，且会偏向"高级"。

✅ 预期结果 / 怎么验证

Claude Code 自动创建项目结构，写出 HTML/CSS/JS，约几分钟完成；浏览器打开即见雏形，已经比多数付费模板好看。

04让它给自己挑刺并改进▾

🎯 这步要干嘛

用 design review skill 让 Claude Code 截图自评、自动修正薄弱处。

📌 操作 / 命令

Review this design and improve it.
🧠 费曼 · 为什么这么做

这步是 design review skill 的主场，也是最反直觉的一点：AI 评审并修正自己的作品。它会给当前站点截图，像挑剔的设计师一样列问题——英雄区间距可以更紧、CTA 按钮对比度不够、菜单区视觉层级要加强——然后自动改掉。这把"生成一次就完"变成了"生成→评审→迭代"的闭环，质量台阶就是这么上去的。

✅ 预期结果 / 怎么验证

刷新页面能看到明显差异：间距更干净、按钮更跳、整体更"有意图"。

05加入动效（motion design）▾

🎯 这步要干嘛

用一条 prompt 加渐入 / 漂浮 / 悬停动效。

📌 操作 / 命令

Add subtle animations — fade in the hero on load,
make the coffee beans drift slowly across the background,
and add a hover effect on the menu cards so they lift and tilt.
🧠 费曼 · 为什么这么做

动效是"$500 的站"和"$10,000 的站"的分界线之一。注意 prompt 里每个动效都说清了触发时机 + 对象 + 行为（加载时英雄区渐入、背景咖啡豆缓慢漂移、卡片悬停抬起倾斜）。描述得越具体，它写出的动画越贴近你脑中的画面，而不是随便加几个 transition。

✅ 预期结果 / 怎么验证

刷新后：英雄区优雅渐入、咖啡豆漂浮、菜单卡片悬停时抬起——页面"活"了起来。

06光标物理交互（进阶）▾

🎯 这步要干嘛

让背景咖啡豆实时响应鼠标位置，像在液体里被推开。

📌 操作 / 命令

Make the coffee beans respond to the mouse —
they should gently push away from the cursor as it moves,
like they're floating in liquid.
🧠 费曼 · 为什么这么做

这是进阶的实时交互：Claude Code 用鼠标位置追踪 + 一点点物理（距离衰减的斥力）写出 JavaScript。值得学的是——你依然只描述了"想要的体感"（像漂在液体里被推开），把"用什么算法实现"留给它。复杂效果不需要你懂实现，只需要你能把"感觉"讲清楚。

✅ 预期结果 / 怎么验证

刷新后移动鼠标，咖啡豆会从光标处轻轻推开，效果接近"有生命感"，会让访客停下来盯着看。

07上线到真实域名▾

🎯 这步要干嘛

把站点打包成静态文件，上传到托管并绑定域名。

📌 操作 / 命令

# 在 Claude Code 里让它打包：
Bundle the site into a single folder ready for static hosting.
# 它会把 HTML/CSS/JS/资源 打成一个可直接静态托管的文件夹
# 然后：托管商后台 → 文件管理器 → 把文件拖进 public_html/ → 绑定域名
🧠 费曼 · 为什么这么做

静态站（纯 HTML/CSS/JS、无后端）可以直接丢进任何静态托管的 public_html 目录就能跑——这就是为什么让 Claude Code "bundle 成单个文件夹"：它把零散文件理成一个可直接上传的成品包，上线只剩"拖文件 + 绑域名"两个体力活。托管商不重要，视频用 Hostinger 是因为它有带货折扣，你换 Netlify/Cloudflare Pages/Vercel 等免费方案同样可行。

✅ 预期结果 / 怎么验证

上传完成、绑定域名后，访问你的域名即可看到上线的网站。

⚠️ 常见坑

视频这段是 Hostinger 赞助/带货（含 affiliate 折扣链接），别被"必须用它"误导——这步的本质只是"静态文件丢上托管"。

🧠费曼总纲重讲：把 Claude Code 当"会自我评审的装修队"▾

把整个流程想成装修一套样板房：

  • Claude Code = 施工队：手很巧，给图就能盖，但默认没什么审美。
  • 两个 design skill = 请来的室内设计总监：aesthetic skill 是"高级感的审美标准"，design review skill 是"验收时拿着清单挑刺并返工"。装上它俩，施工队就从"包工头"升级成"有设计总监盯着的精装团队"。
  • 你的 prompt = 给设计总监的需求简报：你不画施工图，你描述"我要温暖、沉稳、杂志感、暗色配奶油金"。简报越讲清"感觉"，成品越对味。
  • "review and improve" = 装修验收会：设计总监自己拍照、对照标准找毛病、当场返工。这一轮闭环是质感的来源。
  • 动效 / 光标交互 = 软装与灯光：硬装好了之后加的"让空间活起来"的细节，依然只描述体感，实现交给施工队。
  • 打包上线 = 把样板房搬到真地址：静态文件就是可整体搬运的成品，拖进托管、挂上门牌（域名）就开门迎客。

一句话内核：你负责把"想要的感觉"讲清楚，Claude Code（带着审美 skill）负责把成百上千个实现决策做对，并自己验收返工。

🎁最终成品 / 验收清单▾

照着做完，你应该得到：

  • ✅ 一个暗色 + 奶油金、杂志感的高端落地页，含英雄区 / 溯源 / 招牌菜单 / 联系四个版块；
  • ✅ 加载渐入、背景元素漂浮、卡片悬停抬起的动效；
  • ✅ 背景元素随光标实时被推开的物理交互；
  • ✅ 打包好的静态文件，已上线在你自己的域名上。
怎么确认达成

用手机和电脑各打开一次你的域名：动效流畅、光标交互生效、各版块在小屏不塌、访客看不出是 AI 做的——就算过关。

✦5 条要点▾

  1. skill 是分水岭：让 Claude Code 出"设计"而非"代码"的关键，是 aesthetic + design review 两个 skill，不是更长的 prompt。
  2. 描述感觉，不描述代码：给 mood / 配色 / 版块 / 调性，把实现决策留给它。
  3. "review and improve" 值钱：生成→自评→返工的闭环，是质感的真正来源，别省这一步。
  4. 复杂效果只需讲清体感：连光标物理交互都能用一句"像漂在液体里被推开"拿下。
  5. 上线是体力活，托管可替换：静态站打包后丢任意托管即可，别被赞助绑定到某家。
